What makes a good UI? I like to explore the edges of how User Interface can work change experiences and while I like to explore out of the world 3D experiences created with Virtual Reality, I also like to find elegance in retro designs. So I wanted to create a design that resonates with retro + textureless 3d feel. As opposed to modern UI elements floating on top of our perceived background, neumorphic components pretend to extrude from the background.

The beauty of neumorphic surfaces in cards

As much fun it is to deal with the flat and minimal UI design patterns, there is an attraction in curved surfaces like imperfect paper. When put in a Swift UIView, and given positive and negative shawows with a slight contrast, there is a fresh feel to the whole experience. These cards do not flow like the modern cards, and instead feel like most of the card's back lies falt on surface with only slightly curved edges.

Challenges faced in user testing

  1. The adaptability of this trend is yet to be tested extensively. Feedback from my users so far has informned me of mixed reviews: with some users being really excited about this new UI, while others could not care less.
  2. The design seems technically feasible in web and iOS platforms, but I have to yet test it on Android and non-native plaforms./li>
  3. The lack of contrast flat out annoyed some of my users and this leads me to think that this might not be an accessible design. When doing accessibility test, this current design flat out failed and has to be changed drastically in order to be usable.

Tech stack

To test this idea out on a real mobile app, I created a basic app composing of different components: concave and convex cards, buttons, button images and segments controls. I acheived these by manipulating the shadows within and around each of these components. Using Xcode and Swift 5, and extensions of UIButton, UILabel and UIView, I was able to achieve these special effects. Check out my open source Github code here.

Learning: User experience

As I worked with several apps in myriad domains such as gaming, dating, social media, health, I learned that each of these apps have purposes and hence needed to use different solutions to match exact requirements. As I navigated between different apps, I started to notice the fine tunings in the context of deeplinking. I would work with the client teams to in turn understand the user needs. During this process, I noticed the fine details when it comes to functionality, and how it can affect user experience irrespective of the visual design. The UX learnings from Branch stayed with me even when I left Branch to pursue my Masters degree and documented my take on Amazon Prime Video as a user experience


Takeaway

Neumorphism could be the future of UI design. As of January 2020, it seems fresh and new, it does come with potential caveats and has to be dealt with caution. Going forward, I would love to explore this pattern more by performing usability testing and user feedback.
